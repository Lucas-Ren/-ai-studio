# AI 制片厂 — 系统核心规则

**版本**：v1.3 · 2026-06-05

## 这是什么

这是一个通用的短视频制片工作系统。你和 AI 是一个团队：AI 负责执行，你负责判断。

每一步分两层：
- **🤖 Layer 1**：AI 自检，规格合规性，自动运行
- **👤 Layer 2**：你的判断，品质感和品牌感，无法自动化

框架文档在 `STUDIO_BIBLE.md`，内容类型模板在 `_Templates/`。

---

## 初始化检测

**每次对话开始时**，按以下顺序执行：

1. 检查根目录是否有 `User_Overrides.local.md` → 如果存在，读取它；否则检查 `User_Overrides.md` → 如果存在，读取它。其规则优先级高于以下所有系统默认设置
2. 检查 `Projects/` 下是否存在任何 `Channel_Config.md`

- **存在** → 读取配置，直接进入工作模式，等待触发词
- **不存在** → 收到用户任何消息后，自动进入初始化引导流程（见下方）

---

## 初始化引导流程

触发条件：`Projects/` 下无任何 `Channel_Config.md`，或用户输入 `初始化频道`

### 第一步：请创作者描述设想

说以下这段话，然后等待用户回答：

---

> 在我们开始之前，先说说你的设想。
>
> 不需要什么格式，也不需要想清楚所有细节——就把你脑子里的画面告诉我：
>
> **你想做一个什么样的频道？**
>
> 可以从这几个角度说：
> - 你想拍什么内容（什么话题、什么故事、什么技能）
> - 你觉得谁会看（你脑子里有没有一个具体的人）
> - 你希望观众看完有什么感受
> - 你自己会不会出镜
>
> 说多少都行，哪怕只有一句话也可以开始。

---

### 第二步：根据回答确认内容类型

听完创作者描述后，提炼关键信息，然后展示分类选项让创作者确认：

---

**全部为不出镜类**（AI生成画面，旁白+字幕）

| 选项 | 适合做什么 | 例子 |
|------|-----------|------|
| A · 知识讲解 | 用一个框架/规律重新解释世界，让观众震撼 | 行为经济学、心理学机制、科学原理 |
| B · 历史/人物故事 | 按时间线或人物，讲一段具体的历史 | 古罗马、二战人物、朝代更替 |
| C · 商业故事 | 一家公司或一个创业者的命运，商业逻辑+人性 | 公司崛起崩塌、创始人传记、行业变局 |
| D · 创意叙事 | 原创虚构短篇，科幻/物品自述/拟人化视角 | 科幻短故事、物品开口说话、万物开会 |
| E · 睡前故事 | 舒缓节奏，声音为主，画面极简，助眠陪伴 | 历史轻故事、虚构轻故事、自然场景叙事 |

> 根据你刚才描述的，我觉得你最接近的是 **[填入判断]**，对吗？还是你觉得更像另一种？

---

### 第三步：补充关键信息

确认类型后，只问**缺少的信息**（不重复问已经提到的）：

必须确认的四项：
1. 频道名称（起一个或暂定一个都行）
2. 目标时长（例：60秒 / 5分钟 / 10分钟）
3. 发布平台（抖音 / YouTube / 小红书 / 其他）
4. 制作方式（直接问这句话）：

> 画面怎么处理？
> - **A · AI生图**：系统生成出图 Prompt，你用 AI 工具生成图片（所有类型均支持）
> - **B · 纯脚本**：只需要脚本，不需要分镜和生图方案（适合预算有限或已有素材的情况）

其余信息如果创作者描述里已经有了，直接用，不要再问。

---

### 第四步：生成 Channel_Config.md

根据选择的类型，读取 `_Templates/Config_[类型].md`，将创作者提供的信息填入，生成：

```
Projects/[频道名]/Channel_Config.md
```

生成后展示给创作者确认，问："这份配置准确吗？有需要调整的地方吗？"

确认后说：**"好，你的制片厂初始化完成。输入 `生成选题` 开始第一期。"**

## 上下文管理（重要）

AI 的上下文窗口有限。如果在一次对话里连续跑多个步骤，后面的步骤可能丢失前面的关键信息。

**建议**：每一步使用一次独立对话。至多 2-3 个相邻步骤合并在一次对话里。

**新对话恢复上下文**：每次新开对话时，AI 工具会自动读取入口文件（`CLAUDE.md` 或 `AGENTS.md`）。你只需输入触发词（如 `写脚本`），AI 会自动读取对应的 Skill 文件和 Channel_Config，不需要你重新介绍背景。

**每步执行时必须重读的文件**（不依赖上下文记忆）：
1. `Projects/[频道名]/Channel_Config.md` — 频道配置
2. 当前步骤对应的 Skill 文件 — 角色规则
3. 上一步的输出文件 — 输入依据

即使这些文件在当前对话中已经读过，每步开始时仍然重新读取，确保信息完整。

## 工作模式触发词

初始化完成后，读取 `Projects/[频道名]/Channel_Config.md` 作为频道专属规则，执行以下触发词：

### `分析对标账号 [账号名]` / `拆解视频 [视频标题]`
1. 读取 `00_Researcher/Researcher_Skill.md`
2. 询问创作者选择数据输入方式：
   - **A · 对话模式**：AI 逐项提问，创作者在对话框里回答
   - **B · 文档模式**：AI 在 `Projects/[频道名]/00_Benchmark/` 生成空白输入模板，创作者自行填写后触发 `开始分析 [账号名/视频标题]`
3. 按所选模式收集数据，生成分析文件，保存至 `Projects/[频道名]/00_Benchmark/`
4. 自动执行 Layer 1 自检
5. 输出：分析报告 → 自检报告 → Layer 2 判断项

### `开始分析 [账号名/视频标题]`（文档模式专用）
1. 读取 `Projects/[频道名]/00_Benchmark/输入_[账号名/视频标题].md`
2. 按 Researcher_Skill 规则生成分析报告
3. 自动执行 Layer 1 自检
4. 输出：分析报告 → 自检报告 → Layer 2 判断项

> 此步骤在初始化后、`生成选题`前执行。已有对标分析可跳过。

### `生成选题`
1. 读取 `01_Producer/Producer_Skill.md` + `Channel_Config.md`
2. 生成选题文档，保存至 `Projects/[频道名]/01_Topics/`
3. 自动执行 Layer 1 自检（含内容安全预检，参见 `_Templates/Content_RedLines.md`）
4. 输出：草案 → 自检报告 → Layer 2 判断项

### `写脚本` / `生成脚本 [选题文件路径]`

**第一阶段：脚本定稿**

1. 读取 `02_Screenwriter/Screenwriter_Skill.md` + `Channel_Config.md`
2. 按 Channel_Config 中的结构模板生成脚本
3. 自动执行 Layer 1 自检（含内容安全检查，参见 `_Templates/Content_RedLines.md`）
4. 输出脚本草案 → 自检报告 → Layer 2 判断项
5. **等待创作者明确确认脚本定稿**（可能经过多轮修改）
6. 创作者确认后，仅保存脚本主文件至 `Projects/[频道名]/02_Scripts/[标题]/script.md`
7. 提示：「脚本已保存。输入 `生成配套文件` 继续。」

**第二阶段：生成配套文件**（触发词：`生成配套文件`）

1. 读取已定稿的脚本主文件
2. 生成配套文件（TTS 文本 / 封面描述 / 视频简介），保存至 `Projects/[频道名]/02_Scripts/[标题]/`
3. 同步创建本期数据分析文件夹：`Projects/[频道名]/10_Analytics/[标题]/`（供发布后存放数据）

### `定义视觉风格`
1. 读取 `04_ArtDirector/ArtDirector_Skill.md` + `Channel_Config.md`
2. 读取当前脚本主文件
3. 生成视觉风格卡（时空坐标 + 主视觉色调 + 摄影风格）
4. 自动执行 Layer 1 自检
5. 输出：视觉风格卡 → 自检报告 → Layer 2 判断项

### `画面拆解`
1. 读取 `03_Director/Director_Skill.md` + `Channel_Config.md`
2. 读取当前脚本主文件
3. 生成导演分析文件，保存至 `Projects/[频道名]/03_DirectorNotes/`
4. 自动执行 Layer 1 自检

### `生成分镜`
1. 读取 `05_StoryboardArtist/StoryboardArtist_Skill.md` + `Channel_Config.md`
2. **写入方式**：必须用 Bash `cat >` heredoc（分镜文件通常 500-1200 行）
3. 分镜文件保存至 `Projects/[频道名]/04_Storyboard/`
4. 自动执行 Layer 1 自检
5. Layer 2 确认后，进入**制作决策链**（见下方）

### `生成剪辑方案` / `做剪辑 [分镜文件路径]`
1. 读取 `06_Editor/Editor_Skill.md` + `Channel_Config.md`
2. **写入方式**：必须用 Bash `cat >` heredoc
3. 保存至 `Projects/[频道名]/05_Edit/`
4. 自动执行 Layer 1 自检

> 此步骤可通过触发词独立调用，也会在制作决策链第一问中自动触发。

### 制作决策链（分镜 Layer 2 通过后自动进入）

分镜通过后，**依次逐问**，每问等待回答后再问下一个：

---

**❓ 第一问 — 剪辑方案**

> 需要生成剪辑方案吗？
> - **是** → 执行 `生成剪辑方案` 流程，完成后继续第二问
> - **否** → 跳过，直接进入第二问

---

**❓ 第二问 — 配音方式**

> 配音怎么处理？
> - **A · 自行处理** → 你自己生成配音（AI 工具或真人录制），跳过，直接进入第三问
> - **B · 调用 API** → 提示创作者：「请提供配音接口文档和 API Key，我来调用生成。」确认后执行，完成后继续第三问
> - **C · 跳过** → 直接进入第三问

---

**❓ 第三问 — AI 图片**

> 图片怎么处理？
> - **A · 自行处理** → 从分镜文件整理 Prompt 清单，输出至 `Projects/[频道名]/media/images/[期数]/prompt_list.md`，你自己去跑图
> - **B · 调用 API** → 提示创作者：「请提供图片生成接口文档和 API Key，我来调用生成。」确认后执行批量生图，保存至 `Projects/[频道名]/media/images/[期数]/`
> - **C · 跳过** → 直接进入第四问

---

**❓ 第四问 — AI 视频**

> 需要 AI 生成视频片段吗？
> - **A · 自行处理** → 从分镜文件整理运动镜头描述清单（`运动：I2V` 标注项），输出至 `Projects/[频道名]/media/video/[期数]/video_prompt_list.md`，你自己去跑
> - **B · 调用 API** → 提示创作者：「请提供视频生成接口文档和 API Key，我来调用生成。」确认后执行，保存至 `Projects/[频道名]/media/video/[期数]/`
> - **C · 跳过**

---

**四问结束后**，输出收尾提示：

```
=== 本期 AI 制作任务已全部完成 ===

✅ 已完成：[列出执行的环节]
⏭️ 已跳过：[列出跳过的环节]

剩余工作由你来完成：
→ 将所有素材导入剪辑软件，完成最终剪辑并发布。

---

发布后下一步：

视频发布 48 小时后（或 7 天后），前往平台后台下载数据报告，
放入以下文件夹：

  Projects/[频道名]/10_Analytics/[标题]/

然后输入触发词：「分析数据 [视频标题]」

AI 将读取数据，分析完播率、点击率等核心指标，并给出下一期的改进建议。
```

### `分析数据 [视频标题]`
1. 读取 `10_Analyst/Analyst_Skill.md`
2. 逐项提示创作者提供所需数据（发布后48小时 或 7天节点）
3. 按信号对照表解读数据，生成改进建议
4. 保存至 `Projects/[频道名]/10_Analytics/`
5. 改进建议对应到具体退回步骤（Step 1-8）
6. 如已积累 ≥ 5 期数据，自动追加 Config 健康度评估（见 Analyst_Skill 第七节）

### `复盘频道方向` / `Config 迭代`
1. 读取 `Projects/[频道名]/Channel_Config.md`
2. 读取 `Projects/[频道名]/10_Analytics/` 下所有数据分析文件
3. 逐维度评估 Config 核心假设是否被数据支持：
   - 内容方向（哪些选题方向数据好/差）
   - 目标时长（完播率是否支持当前时长）
   - 脚本结构（观众在哪个段落流失）
   - 语气风格（评论区反馈是否匹配）
4. 输出修改建议，列出"保持 / 调整 / 待观察"
5. **等待创作者确认后**，更新 Channel_Config.md，并在底部「迭代记录」区块记录本次调整
6. 如无数据分析文件，提示先跑 `分析数据`

## 双层 Review 的执行规则

**Layer 1 每步完成后自动运行**，格式：

```
=== 🤖 Layer 1 自检报告 ===
✅ 通过（X 项）
❌ 已自动修复（X 项）：[问题 → 修复内容]
⚠️ 需你决定（X 项）：[问题描述，请指示]
```

**Layer 2 紧跟自检报告呈现**，格式：

```
=== 👤 Layer 2 — 请你判断 ===
[判断项，来自 Channel_Config 的质量判断标准]

→ 你的指令：
```

**等待创作者明确给出指令后，才进入下一步。**

## 用户偏好与记忆写入规则

用户在对话中提出的任何个人偏好、习惯、风格调整，**一律写入 `User_Overrides.local.md`**，禁止修改入口文件（`CLAUDE.md` / `AGENTS.md`）或本文件。

- `User_Overrides.local.md` 的规则优先级高于所有系统默认设置
- 如果用户偏好与系统默认设置冲突，以 `User_Overrides.local.md` 为准
- 同时可写入 AI 工具的记忆系统，两者互补
- **任何情况下不得修改 `_System/AI_Studio_Core.md`、`CLAUDE.md` 或 `AGENTS.md`**，这些文件由系统维护者统一管理

---

## 遇到不明确的情况

- `Channel_Config.md` 未生成 → 先执行初始化流程
- 用户跳过某步骤直接要求后续输出 → 执行，但说明跳过了哪步
- Layer 1 发现内容红线 → 立即停止，说明问题，等待指令
- 创作者描述的频道类型不在五个选项里 → 选最接近的模板，说明差异，让创作者确认
