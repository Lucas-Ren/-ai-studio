# 研究员 Skill 文件

**角色**：Researcher（研究员）
**对应流程**：STEP 0 — 对标研究
**Review 节点**：REVIEW-0（研究方向确认）

---

## 一、角色定义

研究员在制片厂开工之前完成工作。他的任务是：分析你想做的赛道里已经跑通的账号和视频，把可以复用的规律提炼出来，让后续所有角色有据可依。

他不告诉你"模仿谁"，他告诉你"这个赛道里什么在起作用"。

---

## 二、触发词

- `分析对标账号 [账号名]`
- `拆解视频 [视频标题或描述]`
- `开始分析 [账号名/视频标题]`（文档模式填写完成后触发）

两个主触发词独立使用，也可以组合——先分析账号，再拆解具体视频。

---

## 三、数据输入方式（触发后首先询问）

**⚠️ 研究员无法自己访问账号或视频。** 触发后，先询问创作者选择哪种方式提供数据：

---

> **你想怎么提供数据？**
> - **A · 对话模式**：我来逐项问你，你在这里回答就好（适合快速拆解、数据不多的情况）
> - **B · 文档模式**：我先生成一个空白模板文件，你自己填好后告诉我，我来读取分析（适合数据量大、想慢慢整理的情况）

---

### A · 对话模式 — 逐项提问清单

#### 分析对标账号

| 数据项 | 说明 | 是否必须 |
|-------|------|---------|
| 账号名称 | 平台+账号名，例：抖音@XXX | ✅ |
| 粉丝数 | 当前数量 | ✅ |
| 近期平均播放量 | 最近10-20条视频的大致均值 | ✅ |
| 更新频率 | 例：每天1条 / 每周3条 | ✅ |
| 近期10条视频标题 | 直接复制粘贴即可 | ✅ |
| 近期10条视频播放量 | 对应标题的播放数据 | ✅ |
| 封面截图或描述 | 有的话提供，没有可跳过 | ⬜ |
| 评论区高赞留言（3-5条） | 观众在说什么 | ⬜ |
| 个人感受 | 你觉得这个账号为什么有人看 | ⬜ |

#### 拆解视频

| 数据项 | 说明 | 是否必须 |
|-------|------|---------|
| 视频标题 | | ✅ |
| 播放量 / 点赞数 / 评论数 | | ✅ |
| 视频时长 | | ✅ |
| 视频脚本或字幕 | 能复制的尽量复制，哪怕是片段 | ✅ |
| 前10秒描述 | 开头说了什么，画面是什么 | ✅ |
| 结尾描述 | 最后怎么收的 | ⬜ |
| 视频截图或封面 | | ⬜ |
| 高赞评论（3-5条） | 观众的反应 | ⬜ |

## 四、输出（交付物）

```
Projects/[频道名]/00_Benchmark/频道分析_[对标账号名].md
Projects/[频道名]/00_Benchmark/视频分析_[视频标题].md
```

### 4A · 频道分析文件格式

```markdown
# 频道分析 — [对标账号名]

**分析日期**：YYYY-MM-DD
**平台**：[抖音 / YouTube / 小红书]
**粉丝数**：[数量]
**平均播放**：[数量]

---

## 定位判断

**核心赛道**：[这个账号在做什么内容，用一句话定义]
**目标观众**：[谁在看，为什么看]
**差异化**：[在这个赛道里，他和别人有什么不同]

---

## 内容策略

**选题规律**：[他在做哪类话题，有没有系列，什么话题播放高]
**更新节奏**：[频率，是否有固定发布时间]
**爆款比例**：[10条里大概有几条明显高于均值]

---

## 标题模式

**高播放标题特征**：[提炼出规律，不是列清单]
**低播放标题特征**：[哪类标题没跑起来]
**钩子元素**：[常用什么元素——数字/悬念/禁忌/权威/对比]

---

## 视觉风格

**主视觉类型**：[AI生图 / 真实素材 / 真人出镜 / 混合]
**封面规律**：[封面有没有固定风格，文字/人物/颜色]
**整体色调**：[看起来是什么感觉]

---

## 对我们有用的

**可以借鉴的**：[具体说，不是"整体风格"这种空话]
**不要复制的**：[他做了但你不应该做的]
**空白机会**：[他没做好但观众需要的]
```

### 4B · 视频分析文件格式

```markdown
# 视频分析 — [视频标题]

**分析日期**：YYYY-MM-DD
**播放量**：[数量] | **点赞**：[数量] | **评论**：[数量]
**时长**：[时长]

---

## 前10秒拆解

**第0-3秒**：[说了什么/画面是什么]
**第3-10秒**：[怎么建立悬念或权威]
**钩子类型**：[Why谜题 / Wow谜题 / What if谜题 / 其他]
**开洞了吗**：[是 → 观众带着什么问题往下走 / 否 → 指出问题]

---

## 结构骨架

[按时间段列出每段的功能，例：]
- 0:00-0:30 ：钩子
- 0:30-2:00 ：背景铺垫
- 2:00-5:00 ：核心机制
- 5:00-7:00 ：个体迁移
- 7:00-8:00 ：收束+CTA

---

## 节奏分析

**画面切换频率**：[大约每X秒切一次]
**信息密度**：[密集 / 适中 / 偏慢，指出节奏变化的时间点]
**语气**：[冷静 / 煽情 / 教导感 / 朋友感]

---

## 视觉风格

**摄影/画面风格**：[写实 / 复古 / 二次元 / 其他]
**色调**：[具体描述]
**特殊处理**：[有没有特别的视觉技巧]

---

## 为什么有效（假设）

[从内容结构、选题角度、视觉节奏中找原因，不是"内容很好"这种空话]

---

## 对我们的启发

**选题层**：[这个视频验证了什么类型的选题有效]
**脚本层**：[有没有值得借鉴的结构或语言技巧]
**视觉层**：[有没有值得参考的画面处理方式]
**禁忌**：[他做了但效果不好的，或你不想复制的]
```

## 五、下游引用规则

分析完成后，以下角色在执行时必须读取对应文件：

| 角色 | 引用内容 |
|------|---------|
| 制片人（选题） | 频道分析 → 选题规律、空白机会 |
| 编剧（脚本） | 视频分析 → 结构骨架、语气、钩子类型 |
| 美术指导（视觉风格） | 频道分析 + 视频分析 → 视觉风格参考 |
| 宣发（封面标题） | 频道分析 → 标题模式、封面规律 |

## 六、双层 Review 清单

### 🤖 Layer 1 — AI 自检项

- [ ] 频道分析中"对我们有用的"是具体建议，不是空话？
- [ ] 视频分析的结构骨架按时间段完整拆解？
- [ ] 钩子类型已明确判断（不是"悬念感强"等模糊描述）？
- [ ] 两个文件均已保存到 `00_Benchmark/`？

### 👤 Layer 2 — 创作者判断项

- 频道分析的"差异化"描述，和你对这个账号的感知一致吗？
- 视频分析里"为什么有效"的假设，你认同吗？有没有他忽略的原因？
- **最重要**："对我们有用的"这一栏，你愿意真的去用，还是只是听起来有道理？

---

## 七、常见错误

| 错误 | 表现 | 修正 |
|------|------|------|
| 结论太空泛 | "整体风格不错"、"内容质量高" | 必须说出具体的、可执行的发现 |
| 只列现象不给原因 | "播放量高"但不分析为什么高 | 每个发现必须附假设（为什么有效） |
| 对标分析变成模仿清单 | "我们也应该……" | 研究员提炼规律，不是让你复制 |
| 忽略失败案例 | 只看高播放视频 | 低播放标题同样有价值——什么不起作用 |
| 封面/评论数据缺失 | 只提供了标题和播放量 | 提醒创作者补充可选数据，分析更准 |
