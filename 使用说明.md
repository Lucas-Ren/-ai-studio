# AI 制片厂 — 新用户使用说明

**版本**：v1.1 · 2026-06-03

---

## 这是什么

这是一个用 AI 辅助制作短视频的完整系统。你负责判断和决策，AI 负责生成和执行。

系统覆盖从"想做什么频道"到"发布后数据分析"的全流程，分 10 个角色、分步决策链。每一步 AI 先自检（规格、格式、内容安全），再由你做品质判断，确认后才进入下一步。

每步完成后，你会看到这样的输出格式：

```
=== 🤖 Layer 1 自检报告 ===
✅ 通过（5项）
❌ 已自动修复（1项）：标题超字数 → 已压缩至18字

=== 👤 Layer 2 — 请你判断 ===
听到这个选题，你的第一反应是"为什么？！"还是"哦"？

→ 你的指令：
```

看到 `→ 你的指令：` 就是在等你说话。直接输入"通过"进入下一步，或者说"不通过，问题是……"让 AI 修改。

## 第一步：环境准备

### 你需要安装的工具

- **Claude Code**（命令行工具）：[claude.ai/code](https://claude.ai/code)
- **Node.js**（Claude Code 依赖）


安装完成后，打开终端，进入本文件夹：

```
cd 你收到的文件夹路径/11_AI_Studio
claude
```

Claude Code 启动后，它会自动读取 `CLAUDE.md`，加载制片厂规则。你不需要做任何额外配置。

---

## 第二步：初始化你的频道

**第一次启动时**，系统会自动发现 `Projects/` 下没有任何频道配置，进入引导流程，问你三件事：

1. **你想做什么频道？**（内容方向、目标观众、希望观众看完什么感受）
2. **内容类型**：AI 会给你五个选项（知识讲解 / 历史/人物故事 / 商业故事 / 创意叙事 / 睡前故事），你选一个或说"更像……"
3. **四个关键参数**：频道名、目标时长、发布平台、画面素材来源（AI生图 / 纯脚本）

AI 整理后生成一份频道配置文件，展示给你确认。确认后，制片厂初始化完成。

> **注意**：如果你想重新初始化，输入 `初始化频道` 即可重新走一遍引导流程。

---

## 第一期试跑建议

如果你之前已经做过视频，建议用一期已完成的视频来验证系统是否适配你的风格，再正式开工：

1. 按正常流程初始化频道
2. 拿你已有的一个选题，输入 `生成选题` 让 AI 生成一次，和你自己当时的选题对比
3. 再输入 `写脚本`，看 AI 生成的脚本和你已有的脚本差异在哪里
4. 这次试跑的目的不是发布，是**调校系统和你的风格匹配度**
5. 如果发现偏差，在 `User_Overrides.md` 里写下你的调整

这比直接开工更安全——花 30 分钟试跑，省去后续反复修改的时间。

---

## 第三步：正式开工前——分析对标账号（可选但推荐）

在生成第一期选题前，建议先把你找好的对标账号资料输入系统，让 AI 提炼可用规律，为后续选题和脚本提供参考。

触发词：
```
分析对标账号 [账号名]
```
例：`分析对标账号 抖音@半佛仙人`

触发后，系统会先问你选哪种方式提供数据：

**A · 对话模式** — 在对话框里聊，AI 逐项提问，你直接回答。适合快速拆解、数据不多的情况。

**B · 文档模式** — AI 在 `00_Benchmark/` 生成一个空白模板文件，你自己去填，填完后输入 `开始分析 [账号名]`，AI 读取文件出报告。适合数据量大、想慢慢整理的情况。

**分析结果保存到：**
```
Projects/[你的频道名]/00_Benchmark/频道分析_[对标账号名].md
```

### 拆解单条爆款视频

```
拆解视频 [视频标题]
```

同样支持 A · 对话模式 / B · 文档模式，触发后选择即可。

**分析结果保存到：**
```
Projects/[你的频道名]/00_Benchmark/视频分析_[视频标题].md
```

> 如果你有多个对标账号，可以多次使用 `分析对标账号`，所有分析文件都会累积在 `00_Benchmark/` 下，后续选题和脚本生成时系统会一并参考。

## 第四步：生产流水线（7个步骤）

初始化完成、对标分析完成后，按以下顺序推进。**每步都用触发词启动**，不需要手动找文件。

---

### STEP 1 · 生成选题

```
生成选题
```

AI 根据你的频道配置和对标分析，生成选题方案（标题、核心矛盾、30秒开头骨架）。

**你需要判断的：**
- 听到核心问题，你的第一反应是"为什么？！"还是"哦"？
- 你会不会想把这个话题讲给某个具体的人听？

选题文件保存到：`Projects/[频道名]/01_Topics/`

---

### STEP 2 · 写脚本

```
写脚本
```
或指定选题文件：
```
生成脚本 Projects/[频道名]/01_Topics/[选题文件名].md
```

AI 展开完整脚本（仅主文件），经你确认定稿后保存。

**你需要判断的：**
- 从头读一遍，每隔30秒有没有"啊原来如此"的感觉？
- 语气是否冷静旁观，不共情，不说教？

脚本确认后输入 `生成配套文件`，AI 再生成：配音版文本（TTS）、封面描述、视频简介，以及本期数据分析文件夹 `Projects/[频道名]/10_Analytics/[标题]/`。

> 这样设计是为了避免脚本改稿时配套文件也要跟着重做。先定稿，再生成。

---

### STEP 3A · 定义视觉风格

> **关于 STEP 3A / 3B / 3C**：三个子步骤按顺序执行（3A → 3B → 3C），彼此依赖。如果你的制作方式是 **C·纯脚本**（不需要生图），可以跳过 3A 和 3C，只跑 3B 画面拆解作为剪辑参考。

脚本通过后自动触发，或手动：
```
定义视觉风格
```

AI 根据脚本内容确定全片时空坐标（时代、地域）、主色调、摄影风格。

**你需要判断的：**
- 这个视觉风格和你想呈现的情绪是否匹配？

---

### STEP 3B · 画面拆解

```
画面拆解
```

AI 把脚本每句旁白拆成画面单元，标注"具象型（A）"或"抽象/概念型（B）"。

**你需要判断的：**
- 有没有哪个画面单元你脑子里能看见，但 AI 描述的完全不是那个意思？

画面拆解文件保存到：`Projects/[频道名]/03_DirectorNotes/`

---

### STEP 3C · 生成分镜

```
生成分镜
```

AI 把画面单元转化为 T2I 出图 Prompt + 运动指令（分镜表）。

**你需要判断的：**
- 快速翻一遍 Prompt，有没有你脑子里看不到画面的描述？
- Hook 段排出来节奏够不够快？

分镜文件保存到：`Projects/[频道名]/04_Storyboard/`

---

### STEP 4 · 制作决策链（分镜通过后自动进入）

分镜 Layer 2 通过后，AI 依次询问四个问题，每问等待你回答后再问下一个：

**第一问：需要生成剪辑方案吗？**

选"是"时 AI 生成完整时间线方案，选"否"直接跳到第二问。

```
（可单独触发）生成剪辑方案
```

剪辑方案文件保存到：`Projects/[频道名]/05_Edit/`

---

**第二问：配音怎么处理？**

- **A · 自行处理**：你自己生成配音（AI 工具或真人录制），跳过
- **B · 调用 API**：你提供配音接口文档 + API Key，AI 调用生成
- **C · 跳过**

配音文件保存到：`Projects/[频道名]/media/audio/`

---

**第三问：图片怎么处理？**

- **A · 自行处理**：AI 从分镜文件整理 Prompt 清单，输出至 `media/images/[期数]/prompt_list.md`，你自己去跑图
- **B · 调用 API**：你提供图片生成接口文档 + API Key，AI 调用批量生图并保存
- **C · 跳过**

图片文件保存到：`Projects/[频道名]/media/images/[期数]/`

---

**第四问：需要 AI 生成视频片段吗？**（如 Kling、Runway 等）

- **A · 自行处理**：AI 整理分镜中的 I2V 运动描述清单，输出至 `media/video/[期数]/video_prompt_list.md`，你自己去跑
- **B · 调用 API**：你提供视频生成接口文档 + API Key，AI 调用生成并保存
- **C · 跳过**

视频文件保存到：`Projects/[频道名]/media/video/[期数]/`

---

四问完成后，AI 输出收尾提示，告知本期 AI 制作任务已全部完成，剩余工作（剪辑成片）由你来完成。

---

### STEP 5 · 宣发材料（脚本定稿后生成）

封面描述和视频简介在 STEP 2 脚本定稿后，通过 `生成配套文件` 触发生成，保存在：
```
Projects/[频道名]/02_Scripts/[标题]/封面描述_v1.txt
Projects/[频道名]/02_Scripts/[标题]/视频简介_v1.txt
```

如需单独重新生成：
```
生成封面标题
```

## 视频发布后：数据分析

数据文件夹在写脚本阶段已自动创建，视频发布后直接将平台下载的数据放入：
```
Projects/[频道名]/10_Analytics/[视频标题]/
```

然后输入触发词：
```
分析数据 [视频标题]
```

AI 会逐项提示你输入数据（播放量、完播率、点赞、评论等），解读信号，给出具体改进建议（退回哪一步修改）。

积累5期数据后，可以运行：
```
复盘频道方向
```
AI 会评估你的频道配置是否需要调整，列出"保持 / 调整 / 待观察"建议。

## 文件结构总览

```
11_AI_Studio/                    ← 系统层（升级时整包替换）
├── CLAUDE.md                    ← 系统规则（AI自动读取，你不需要动）
├── User_Overrides.md            ← 你的个人调整（升级时不会被替换）
├── _Templates/                  ← 频道配置模板（初始化时使用）
├── 00_Researcher/ ~ 10_Analyst/ ← 各角色规则文件（AI自动读取）
│
└── Projects/                    ← 用户数据层（永远不动）
    └── [你的频道名]/
        ├── Channel_Config.md    ← 频道配置（初始化时生成）
        ├── 00_Benchmark/        ← 对标账号分析文件
        ├── 01_Topics/           ← 选题文件
        ├── 02_Scripts/          ← 脚本文件（含TTS、封面、简介）
        ├── 03_DirectorNotes/    ← 画面拆解文件
        ├── 04_Storyboard/       ← 分镜Prompt文件
        ├── 05_Edit/             ← 剪辑方案文件（可选）
        ├── 10_Analytics/        ← 数据分析文件（写脚本时自动创建子目录）
        └── media/               ← 媒体文件（图片、音频、视频，单独备份）
            ├── images/          ← AI 生成的图片（按期数分子目录）
            ├── audio/           ← TTS 配音文件
            └── video/           ← AI 生成的视频片段（按期数分子目录）
```

### 三层配置优先级

| 层 | 文件 | 谁来管 | 升级时 |
|---|------|-------|--------|
| 系统层 | `_System/AI_Studio_Core.md` + Skill文件 | 系统设计者 | `git pull` 自动更新 |
| 用户覆盖层 | `User_Overrides.local.md` | 你 | gitignored，不动 |
| 频道配置层 | `Channel_Config.md` | 你 | gitignored，不动 |

### 如何升级系统

```bash
git pull
```

就这一步。`User_Overrides.local.md` 和 `Projects/` 已在 `.gitignore` 中，不会被覆盖。

升级完成后查看 `CHANGELOG.md` 了解本次更新了什么。

### 个人调整怎么写

打开 `User_Overrides.local.md`（第一次使用时先把 `User_Overrides.md` 复制重命名为 `User_Overrides.local.md`），按文件里的说明写你的调整。比如：

```
## 脚本风格调整
- 结尾不用反问句，改用陈述句收尾
- 每段必须有一个具体数字或案例
```

AI 每次对话开始时会自动读取这个文件，所有调整立即生效。

> **注意**：`User_Overrides.local.md` 是你的私人文件，已在 `.gitignore` 中，不会随系统更新被覆盖，也不会被上传到 GitHub。

## 每次新开对话的注意事项

Claude 的上下文窗口有限，**建议每个步骤单独开一次新对话**，每步之间最多合并2-3个相邻步骤。

每次新对话开始时，Claude Code 会自动重读 `CLAUDE.md` 和 `Channel_Config.md`，不需要你重新介绍背景。

## 常见问题

**Q：我可以跳过某个步骤吗？**
可以。直接输入后续步骤的触发词，AI 会执行，但会提示你跳过了哪步。建议至少走完选题→脚本→分镜这三步。

**Q：AI 生成的内容我不满意怎么办？**
在 Layer 2 判断环节直接说"不通过，具体问题是……"，AI 会修改后重新提交 Layer 1 自检。

**Q：我有多个频道想同时管理怎么办？**
`Projects/` 下可以有多个频道文件夹，每个频道有独立的 `Channel_Config.md`。每次对话开始时告诉 AI"我要处理的是[频道名]"即可。

**Q：AI 说找不到某个文件怎么办？**
通常是上下文过长导致路径记忆丢失。新开一次对话，重新输入触发词，AI 会重新读取文件。
