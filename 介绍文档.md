# AI 制片厂 — 系统介绍

**版本**：v1.3 · 2026-06-14

---

## 这套系统是怎么来的

做短视频有一个几乎所有人都经历过的困境：

你知道想做什么，但不知道怎么做得稳定。你的第一期也许很好，第三期塌掉了，第七期你甚至不记得自己当初为什么要做这件事。

AI 工具的出现让生成内容变得容易，但同时带来了新的困境：AI 能写出一篇通顺的脚本，但它不知道那篇脚本是否符合你的频道定位，是否和你上一期的风格一致，是否在观众脑子里会留下任何印象。

**AI 制片厂**试图解决的，就是这件事：**让 AI 做它擅长的执行工作，让你做你无法被替代的判断工作**。

---

## 为什么用电影工业的框架

传统电影工业在过去一百年里，解决了一个和短视频创作者完全相同的问题：**如何让一群人按顺序生产出质量可控的故事**。

他们的答案是：角色分工 + Greenlight 机制 + 每日样片 Review + 制片手册。

这套机制的底层逻辑可以直接移植过来。区别只是规模——电影厂有数百人，你的制片厂只有你和 AI。

所以这套系统里，你是**执行制片人（Executive Producer）**。AI 是你雇来的制片人、编剧、导演、美术指导、分镜师、配音、剪辑师、宣发和质量总监。每个角色有自己的专属规则（Skill 文件），有明确的输入和交付物，有清晰的移交节点。

每一步，**AI 先做，你再判断**。AI 不 Greenlight，你 Greenlight。

---

## 系统的核心设计：双层 Review

这是整套系统最重要的设计，也是它区别于"让 AI 直接生成内容"的根本所在。

每一个步骤完成后，都经历两层审核：

**Layer 1 — AI 自检**

AI 自己对照规格清单检查：格式是否正确、关键模块是否齐全、内容是否触碰红线、文件路径是否规范。所有客观可验证的规格项，在这一层消化。你不需要背规则，AI 替你守规则。

**Layer 2 — 你的判断**

Layer 1 通过后，你才需要介入。此时你只需要判断那些 AI 无法判断的事：这个选题听起来让你兴奋吗？脚本的语气是不是你的声音？这期内容发出去，你的老粉会说"这就是我关注你的原因"还是"他最近怎么了"？

**你不需要记任何规则。规则是 Layer 1 的工作。你只需要判断。**

这两层的配合，让系统同时具备两种能力：规格的可靠性（不遗漏、不犯低级错误），和品质的可控性（每一步都经过你的品味过滤）。

## 10 个角色，覆盖从选题到发布的全流程

| 角色 | 负责什么 |
|------|---------|
| **研究员** | 分析对标账号，拆解爆款逻辑，为选题和脚本提供参考 |
| **制片人** | 生成选题，测验"谜题感"，判断选题是否值得做 |
| **编剧** | 展开完整脚本，生成 TTS 配音文本、封面描述、视频简介 |
| **导演** | 把脚本旁白拆解成画面单元，标注画面类型 |
| **美术指导** | 定义全片视觉风格：时空坐标、主色调、摄影风格 |
| **分镜师** | 把画面单元转化为 T2I 出图 Prompt 和运动指令 |
| **配音** | 生成 TTS 停顿标记，推荐配音参数配置；支持 AI 手动、API 调用、真人原声三种路径 |
| **剪辑师** | 生成时间线方案：镜头时长、动画指令、音频轨道规划（可选步骤，按需触发） |
| **宣发** | 生成封面描述、备选标题、视频简介（国内 + YouTube 双版本） |
| **质量总监** | 质检能力内嵌于每步 Layer 1 自检，每步通过即等于全局质检通过 |

发布之后，还有**数据分析师**角色，负责解读发布后数据，给出具体的改进建议，并在积累 5 期以上后评估频道配置是否需要迭代。

## 5 种内容类型，一套通用框架

系统支持五种频道类型：

- **知识讲解**：用框架和规律重新解释世界，AI 生图辅助画面
- **历史/人物故事**：按时间线或人物讲述具体历史，AI 生图还原时代
- **商业故事**：一家公司或创业者的命运，商业逻辑 + 人性
- **创意叙事**：原创虚构短篇，科幻 / 物品自述 / 拟人化视角
- **睡前故事**：舒缓节奏，声音为主，画面极简，助眠陪伴

框架是通用的。各频道的"创作信仰"（语气、选题标准、爆款逻辑）注入在各角色的 Skill 文件里，不在框架层。**换一个频道，换一套 Skill 文件，Studio Bible 和 Review 机制不变。**

## 系统的三层配置

```
_System/AI_Studio_Core.md + Skill 文件  ← 系统层（git pull 自动更新）
User_Overrides.local.md                 ← 你的个人调整（gitignored，永不覆盖）
Projects/[频道名]/Channel_Config.md     ← 频道配置（初始化时生成，持续迭代）
```

**系统层**由系统设计者维护，不需要你手动配置。

**用户覆盖层**是你的私人调整空间。如果你不喜欢某个角色的默认行为——比如脚本结尾风格、选题的数量限制——写进 `User_Overrides.local.md`，立即生效，升级系统时不会丢失。

**频道配置层**是系统理解你的频道的核心依据。它在初始化时通过对话引导生成，记录你的内容方向、目标观众、目标时长、制作方式、风格偏好。每一期数据复盘后，你可以基于数据决定是否更新它。

## 和"让 AI 直接帮我写脚本"的本质区别

直接让 AI 写脚本，得到的是一篇**通顺但无个性**的内容。它不知道你的语气，不记得你上一期讲了什么，不知道你的观众是谁，也无法在三个月后保持一致。

这套系统的价值不在于单次输出质量更高，而在于：

1. **有记忆**：频道配置持久化，每期都在同一套价值观和风格规范里工作
2. **有规则**：Layer 1 替你守住所有可以量化的质量标准
3. **有分工**：你只做判断，不做生成，把时间花在只有你能做的事上
4. **可迭代**：数据反馈 → 配置调整 → 下一期改进，形成闭环

用电影工业的比喻说：这不是找 AI 临时帮你写一个剧本，这是建一个自己的制片厂，AI 是厂里的员工，规则是厂里的制度，你是老板。

## 如何开始

1. 安装 Claude Code（需要 Claude Pro 订阅），进入本文件夹，运行 `claude`
2. 系统自动检测到 `Projects/` 为空，进入初始化引导
3. 用一段话描述你想做的频道，AI 帮你整理成配置文件
4. 确认配置后，输入 `生成选题` 开始第一期

详细操作步骤见 `使用说明.md`。
