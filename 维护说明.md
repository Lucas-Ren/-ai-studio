## 一、哪些文件是你的，哪些文件别动

收到系统后，只有两样东西是你的私有数据，其他所有文件都是系统文件：

```
ai-studio/                           ← 项目根目录（git clone 后得到）
│
│  ── 系统文件（别动，git pull 时会更新）──────────────
│
├── CLAUDE.md
├── AGENTS.md
├── STUDIO_BIBLE.md
├── CHANGELOG.md
├── 使用说明.md
├── 维护说明.md
├── _System/                         ← 系统核心规则
├── 00_Researcher/ ~ 10_Analyst/     ← 所有角色文件夹
└── _Templates/                      ← 频道类型模板
│
│  ── 你的文件（gitignored，升级时永远不动）──────────────
│
├── User_Overrides.local.md          ← 你的个人规则调整
└── Projects/                        ← 你的所有频道内容
    └── [频道名]/
        ├── Channel_Config.md
        └── ...所有产出文件
```

**如果你想调整系统行为**（比如改某个角色的规则、加自己的偏好），正确做法是写在 `User_Overrides.local.md` 里，不要直接改 Skill 文件。原因：Skill 文件会在下次升级时被替换，直接改会丢失。`User_Overrides.local.md` 已在 `.gitignore` 中，升级时永远不会被动。

---

## 二、如何升级

在项目文件夹里运行一行命令：

```bash
git pull
```

就这一步。`User_Overrides.local.md` 和 `Projects/` 已在 `.gitignore` 中，不会被覆盖。

升级完成后，查看 `CHANGELOG.md` 了解这次更新了什么。如果有 ⚠️ 标记的条目，按说明手动处理一下（大多数升级没有 ⚠️ 条目）。

## 三、常见问题

**Q：新开对话后，AI 还记得我的频道吗？**

记得。每次对话开始，Claude Code 会自动读取 `CLAUDE.md`，你只需要输入触发词（如 `写脚本`），AI 会自动找到你的频道配置和上一步的输出文件。不需要重新介绍背景。

如果你有多个频道，建议在触发词后面加上频道名，例如：`写脚本 理性避难所`，避免 AI 读错配置文件。

---

**Q：AI 在对话中途"忘了"前面说的内容怎么办？**

这是 Claude 上下文窗口有限导致的。一次对话里连续跑太多步骤后，后面的步骤可能丢失前面的信息。

解决方法很简单：**每 1-2 个步骤新开一次对话**。每次新对话 AI 都会重新读取所有相关文件，信息完整。

不需要担心"换了对话会忘记"——重要信息都存在文件里，不在对话记忆里。

---

**Q：Claude Code 报错了怎么办？**

几种常见情况：

| 报错类型 | 原因 | 处理方式 |
|---------|------|---------|
| `rate limit` / 速率限制 | API 调用太频繁 | 等 1-2 分钟再试 |
| `context length exceeded` | 上下文溢出 | 新开对话，重新输入触发词 |
| `cannot read file` | 文件路径不存在 | 检查触发词里的文件路径是否正确 |
| 启动时报错 | Node.js 未安装或版本过旧 | 重新安装 Node.js（建议 v18 以上） |

如果报错信息看不懂，把完整报错内容发给我，我帮你判断。

---

**Q：我能不能修改 Skill 文件来调整 AI 的行为？**

技术上可以，但**不建议**。Skill 文件升级时会被整包替换，你的修改会丢失。

正确做法：把你想调整的规则写进 `User_Overrides.md`，它在升级时不会被动。`User_Overrides.md` 支持按角色覆盖（Producer、Screenwriter、Editor 等各自独立），粒度足够细。

---

**Q：我想给频道换一个方向，需要重新初始化吗？**

不需要删东西，直接输入 `初始化频道` 重走一遍引导流程，生成新的 `Channel_Config.md` 覆盖旧的就行。旧的产出文件还在 `Projects/` 里，不会被删除。

如果是同时运营两个不同频道，建议初始化时用不同的频道名，AI 会分别在 `Projects/[频道名]/` 下存档，互不干扰。
