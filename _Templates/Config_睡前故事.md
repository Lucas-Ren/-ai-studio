## 脚本结构模板

```
Settle（定场）
  → 一个时间，一个地方，几句话建立安全感
  → 不是悬念钩子，是邀请入场
  → 例："那是很久以前，一个小镇边上，有一座没人记得名字的图书馆。"

Character（人物）
  → 一个具体的人或生命
  → 不需要深度背景，需要温度
  → 用一个细节让观众感受到他/她是真实的

Journey（旅程）
  → 发生了一些事——不是冲突，是经历
  → 节奏慢，感官细节多：光线、气味、声音、触感
  → 没有敌人，没有危机，只有行进和感受

Rest（归宿）
  → 一切归于平静
  → 主角找到了某种安放：回家、释然、入睡、离开
  → 结尾句应该让听众自然地想闭上眼睛
  → 例："她把书放回书架，灯灭了，雪还在下。"
```

---

## 语气标准

- 语速比其他类型慢 20-30%（在脚本里用较长的句子引导配音节奏）
- 感官描写优先：让听众"看见"和"感受到"，而不是被信息轰炸
- 禁止悬念、惊吓、激烈冲突——这是安全区，不是刺激区
- 禁止说教和总结，结尾不升华，只还原
- 字数控制：每分钟约 200-250 字（比其他类型少，留给声音和停顿空间）

---

## Layer 2 质量判断标准

- Settle：读完开场，你有没有感到"可以放松了"？
- Character：那个人物，有没有一个细节让你觉得他/她是真实的？
- Journey：读的过程中，有没有哪个感官细节让你脑子里出现了画面？
- Rest：最后一句，你愿不愿意就停在这里？
- 整体：深夜一个人听，会不会感到平静？

---

## 角色权重

重度：编剧（语感和节奏决定一切）+ 配音（声音是核心产品）
标准：宣发 + QC
轻度：美术指导（画面极简，氛围为主）+ 分镜师（每段1-2张，低频）+ 制片人 + 导演
