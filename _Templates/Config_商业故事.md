## 脚本结构模板

```
Spark（起点火花）
  → 一个人，一个决定，或一句话
  → 不介绍公司背景，直接进入最关键的那一刻
  → 让观众先看到结果的影子，再回到起点

Build（积累期）
  → 3-5个具体时间节点推进
  → 每个节点是一个决定，不是流水账
  → 用数字和行动说话，不用评价

Crisis（危机点）
  → 最危险、最接近失败的那一刻
  → 短句，留白，让数字和事实说话
  → 不替主角解释，让观众感受压力

Turn（转折）
  → 那个关键选择或意外
  → 可以是英雄决策，也可以是运气，也可以是崩塌
  → 没有正确答案，只有赌注和后果

Legacy（落地）
  → 这个故事说明了什么商业逻辑或人性
  → 冷静陈述，一句话，不感叹，不煽情
  → 让观众自己得出判断
```

---

## 语气标准

- 像在还原一段真实历史，不是在讲励志故事
- 不替主角辩护，不批判主角，呈现选择和后果
- 数字、时间、地点具体，禁止模糊描述
- 结局不一定是成功：崩塌、转型、被收购，同样值得讲

---

## Layer 2 质量判断标准

- Spark：第一句话之后，你想不想继续看这个人怎么了？
- Crisis：那一刻，你有没有感受到压力？
- Turn：转折够不够意外，或者够不够必然？
- Legacy：最后那句话，值不值得截图？
- 整体：看完，你对这门生意或这个人，有没有新的理解？
