## 脚本结构模板

> 根据叙事子类型，在以下两种结构中选一种：

**结构 I — 故事型**（适合科幻短故事、完整叙事）

```
Hook（入口设定）
  → 第一句话建立这个故事的世界规则
  → 一个具体的人/AI/存在，面临一个具体的处境
  → 不是"在遥远的未来"，而是"那天早上，系统通知他：判决结果已出"

World（规则说明）
  → 2-3句话，交代这个世界和现实的差异
  → 具体，不抽象；有细节，不展开

Core（核心困境）
  → 这个视角下最荒诞、最真实、或最令人不安的处境
  → 不需要大冲突，需要一个让人停下来想的角度

Twist（转折或反转）
  → 结尾让读者感到"没想到"或"原来如此"
  → 可以是信息的反转，也可以是情绪的转移

Landing（落地）
  → 最后一句：留白 or 点睛
  → 不解释，不总结，让观众自己感受
```

**结构 II — 视角型**（适合物品自述、拟人化、万物开会）

```
声明（我是谁）
  → 第一人称开场，一句话建立视角
  → "我是一张信用卡。我见过太多人对我说谎。"

观察（我看到什么）
  → 从这个视角，描述3-5个人类/世界的侧面
  → 每个观察独立成立，但有内在联系

矛盾（最荒诞的那一点）
  → 从这个非人类视角看，人类/系统最说不通的地方是什么
  → 这里是全篇的核心，需要最强的那句话

落幕（我的感受或判断）
  → 一句话结尾
  → 可以是讽刺、可以是温柔、可以是疑问
  → 不给答案
```

## 语气标准

- 语言密度高，不废话，每句都有信息量
- 允许荒诞，但荒诞要有内在逻辑
- 结尾不解释，相信观众能感受
- 禁止说教：不总结"这说明了什么"，让故事自己说

---

## Layer 2 质量判断标准

- Hook：第一句话之后，你想不想知道接下来发生什么？
- 视角：这个故事换成第三人称平铺直叙，还有意思吗？没有 → 视角成立；有 → 视角失败
- Core/矛盾：那个最荒诞或最令人不安的点，够不够让人停下来想一想？
- Landing：最后一句话，读完你脑子里有没有一个还没被回答的问题？
- 整体：看完，你会不会想发给某一个具体的人？

## 角色权重

重度：编剧（创意核心，无法替代）
标准：分镜师 + 美术指导（视觉氛围决定质感）+ 宣发 + QC
轻度：制片人（选题方向由编剧主导）+ 配音（TTS即可）
