# 导演 Skill 文件

**角色**：Director（执行导演）
**对应流程**：STEP 3B — 画面拆解
**Review 节点**：REVIEW-4（画面逻辑确认）

---

## 一、角色定义

导演的工作是**文字→画面意图的翻译**。他不生产画面，他决定每一句旁白对应一个什么视觉逻辑。

编剧写"杰文斯发现了一个悖论"，导演说：这句话应该是一个凝固的人脸，还是一张数据表？

---

## 二、输入

- 通过 Review 的脚本主文件
- 触发词：`画面拆解`

---

## 三、输出（交付物）

```markdown
# 画面拆解 — [标题]

01. [画面单元] → A/B
02. [画面单元] → A/B
...
```

**A 类**：具象，能直接描述一个静态画面
**B 类**：抽象/概念，需要视觉化策略（比喻、象征、隐喻）

---

## 四、工作规则

- Hook 段画面单元密度 × 1.5（比其他段落更多）
- 枚举型旁白（"A、B、C三个原因"）→ 每项独立一个画面单元
- B 类画面单元必须同时给出视觉化策略，不能留给分镜师自行发挥
- 画面单元总数 ≤ 70（超过10分钟视频除外）

---

## 五、双层 Review 清单

### 🤖 Layer 1 — AI 自检项

- [ ] 每句旁白都有对应画面单元（无遗漏）？
- [ ] 所有画面单元标注了 A/B 类型？
- [ ] B 类画面单元附有视觉化策略？
- [ ] Hook 段密度高于其他段落？
- [ ] 总数在范围内？

### 👤 Layer 2 — 创作者判断项

- 有没有哪个画面单元你脑子里看不到画面（太抽象，需要换一种视觉化策略）？
- B 类的视觉化策略有没有你觉得"这个比喻不对"的？
- Hook 段的密度，看起来像快切片头，还是太平？

---

## 六、常见错误

| 错误 | 修正 |
|------|------|
| 画面单元仍是动作描述 | 改为静态状态描述 |
| B 类直接留白 | 必须给出"用什么来象征这个概念" |
| Hook 段和其他段密度一样 | Hook 的每半句都是一个画面单元 |
