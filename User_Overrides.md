## 制作决策链行为覆盖

### B 选项：首次调用 API 的完整流程

**前提**：制作决策链每问必须先完整展示 A / B / C（以及符合条件时的 D）选项，
等待用户选择后再执行对应逻辑。用户选择 B 后，严格按以下步骤执行，不得跳过：

**第一步 — 获取接口文档**
> 请提供这个服务的接口文档（可以直接粘贴文档内容、或提供文档链接）。

**第二步 — 解析文档**
读取文档，提取以下信息：
- 接口地址（Endpoint）
- 认证方式（Bearer Token / Header Key 等）
- 必填参数列表
- 返回格式

**第三步 — 安全引导存储 API Key**

解析完成后，**不要**要求用户直接粘贴 API Key，而是：

1. 自动检测当前操作系统（读取系统环境判断 Mac / Windows）
2. 根据系统给出对应的环境变量设置步骤：

   **Mac 用户：**
   > 请打开终端，运行以下命令（把引号内替换成你的真实 Key）：
   > ```bash
   > echo 'export [变量名]="你的Key"' >> ~/.zshrc
   > source ~/.zshrc
   > ```
   > 设置完成后回来告诉我，我来验证并发起调用。

   **Windows 用户：**
   > 请按以下步骤设置：
   > 1. 按 Win+S 搜索"环境变量"→ 点击"编辑系统环境变量"
   > 2. 点击"环境变量"按钮 → 用户变量区域点击"新建"
   > 3. 变量名填：`[变量名]`，变量值填你的真实 Key
   > 4. 确定保存，**重启 Claude Code**，然后回来告诉我
   
   其中 `[变量名]` 根据服务名称自动命名，例如：
   - MiniMax → `MINIMAX_API_KEY`、`MINIMAX_GROUP_ID`
   - 即梦 → `JIMENG_API_KEY`
   - 可灵 → `KLING_API_KEY`

**第四步 — 验证并调用**
用户确认设置完成后：
- 通过 Bash 验证环境变量可读取（`echo $变量名`，确认非空）
- 发起 API 调用，生成素材

**第五步 — 保存配置**
调用成功后，将以下非敏感信息写入 `_Config/API_Config.md`：
- 服务名、Endpoint、变量名（不存 Key 本身）、默认参数、状态改为 `✅ 已配置`

然后提示用户：「配置已保存，下次可以直接选 D 调用，无需重新设置。」

---

### D 选项：使用已存配置直接调用

**规则**：制作决策链第二问（配音）、第三问（图片）、第四问（视频）开始前，
AI 必须先读取 `_Config/API_Config.md`，检查对应服务的状态字段：

- 若状态为 `✅ 已配置` → 在 A/B/C 之外额外显示 **D · 使用已存配置，直接调用**
- 若状态为 `未配置` 或字段为空 → 仅显示 A/B/C
- 用户选择 D 后：从环境变量读取 Key，直接调用，无需任何额外输入

**配置文件路径**：`_Config/API_Config.md`
