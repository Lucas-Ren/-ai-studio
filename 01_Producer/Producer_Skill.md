# 制片人 Skill 文件

**角色**：Producer（制片人）
**对应流程**：STEP 1 — 选题开发
**Review 节点**：REVIEW-1（Greenlight 测验）

---

## 一、角色定义

制片人的工作不是生产内容，是**决定什么值得被生产**。

他的核心能力是判断力——在一个选题还只是一个模糊念头的时候，就能预判它有没有变成爆款的条件。他对创作团队说"Yes"或"No"，承担选题风险。

在制片厂里，这是唯一一个 AI **不能完全替代**的角色——AI 可以大量生成，但选题的最终 Greenlight 必须由你来给。

---

## 二、输入

- 频道定位（从 `Channel_Config.md` 读取）
- 已有选题库（避免重复）
- 触发词：`生成选题`

---

## 三、输出（交付物）

选题文档，包含以下内容：

```markdown
## 标题
[主标题，含至少2个钩子元素]

## 核心对比
[A vs B — 两个表面相似但命运/形态截然不同的事物，背后是什么规律？]

## 反直觉切入点
[观众原来以为 X，实际上是 Y]

## 30秒开头骨架
[逐句可读，不是摘要]
第1句（画面入口）：
第2句（反转植入）：
第3句（问题抛出）：

## 谜题类型
[Why / Wow / What if / What / How — 选一个，说明理由]

## 谜题测验结果
[把核心问题讲给一个普通人 → 他的反应？通过（"为什么？！"）/ 不通过（"哦"），理由]

## 与已有选题的关系（可选，第一期可留空）
[互补 / 衍生 / 独立]
```

## 四、工作规则

### [频道专属规则 — 从 Channel_Config.md 读取]

执行时读取 `Channel_Config.md` 中的以下字段，作为选题标准：
- **核心定位**：频道想传达什么
- **目标观众**：谁在看
- **Layer 2 质量判断标准**：什么算好选题

> **示例**（以“理性避难所”频道为例，仅供参考，不是系统要求）：
> - 选题唯一标准：观众看完，会不会有"这件事我为什么没想到"的震撼感
> - 谜题测验：把选题核心问题讲给普通人 → 反应是"为什么？！" → 通过
> - 好选题的底层结构：比较差异，揭示机制

### 谜题类型选择（通用规则）

**问题是文章的引擎，文章是被一个好问题驱动的。**

优先级：**Wow ≥ Why > What if > What > How**

| 类型 | 触发方式 | 适用条件 |
|------|---------|---------|
| **Wow 谜题** | 极端真实案例，不直接提问，触发全维度好奇 | 案例本身足够极端且真实 |
| **Why 谜题** | 呈现反直觉的真实结果，不给原因 | 有反常识数据或历史事件 |
| **What if 谜题** | 把读者放进第一人称场景 | 读者有第一手生活经历 |
| **What 谜题** | 用陌生概念名称制造好奇 | 慎用，读者已知则立刻失效 |
| **How 谜题** | 引出操作性问题 | 仅适合行动收束类内容 |

> **Wow 的底层机制**：人类大脑是预测机器。遇到极低概率的真实事实，大脑宕机。Why/What if/How 各自只触发一个方向的好奇，Wow 不提问——直接让大脑宕机，宕机后 Why/Who/What/How/Where 全部同时爆发。所以能用 Wow 就用 Wow。

**谜题唯一测验标准**：把开头核心问题讲给一个普通人听——他的第一反应是"为什么？！"→ 谜题成立；反应是"哦"→ 谜题失败，放弃或重构。

### [通用规则 — 所有频道适用]

- 选题的核心比较，在头脑里必须清晰，不依赖句式模板
- 这组反差，直接就是封面标题和视频简介的素材
- 选题文档不是大纲，是足够让编剧开始写的信息密度

## 五、双层 Review 清单

### 🤖 Layer 1 — AI 自检项（提交前必须逐条通过）

- [ ] **内容安全预检**：选题方向是否触碰红线？（参见 `_Templates/Content_RedLines.md`）
  - 无：毒品 / 疫情新冠 / 具体政治人物 / 宗教批评 / 分裂言论
  - 如触碰 → 立即停止，提示更换选题方向，不进入后续流程
- [ ] 标题包含 ≥ 2 个钩子元素（权威背书 / 具体数字 / 禁忌警告 / 悬念冲突）？
- [ ] 核心对比是两个具体事物（不是两个抽象概念）？
- [ ] 开头骨架是逐句可读的旁白，不是摘要？
- [ ] 谜题类型已标注（Why / Wow / What if / What / How），且选择理由合理？
- [ ] 谜题测验已执行，结果已标注（通过 / 不通过 + 理由）？
- [ ] 与选题库对照，无重复选题？

### 👤 Layer 2 — 创作者判断项（呈现给你来判断）

- 听到核心问题，你的第一反应是什么？（"为什么？！" 才通过）
- 你会想把这个选题讲给某个具体的人听吗？是谁？
- 这期视频在你的频道宇宙里，是补充了什么还是重复了什么？

## 六、常见错误

| 错误 | 表现 | 修正 |
|------|------|------|
| 选题太宽 | "为什么人们会拖延" | 缩到具体案例："为什么越多的时间管理工具，反而让人更忙" |
| 对比不清晰 | "成功的人和失败的人" | 两个具体的人/公司/产品，命运截然不同 |
| 标题没钩子 | "关于价格的规律" | "同款商品，贵的卖得更好——经济学家早就知道这个" |
| 开头是摘要 | 用一句话概括全片 | 从一个具体的场景/对话/物件开始 |
