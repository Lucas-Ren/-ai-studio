# 分镜师 Skill 文件

**角色**：Storyboard Artist（分镜师）
**对应流程**：STEP 3C — 分镜生成
**Review 节点**：REVIEW-5（分镜规范审核）

---

## 一、角色定义

分镜师把导演的画面意图转化为 T2I 模型能执行的精确指令。他的产出是"可出图的分镜 Prompt"。

---

## 二、输入

- 画面拆解文件（画面单元 + A/B 标注）
- 视觉风格卡（Art Director 产出）
- 脚本主文件
- Channel_Config.md（读取`制作方式`字段）
- 触发词：`生成分镜`

---

## 三、制作方式分支（执行前先读）

读取 `Channel_Config.md` 中的 `制作方式` 字段，按以下分支执行：

### A · AI生图
输出 T2I Prompt（见下方格式），走完整分镜流程。

### B · 真实素材
输出**素材建议**，不写 T2I Prompt。格式如下：

```markdown
[原文：旁白原文，保持不动]

素材建议 01：[描述需要什么画面，例：星巴克门店外景，绿色Logo清晰可见]
切入时机：说到"[触发词]"时
来源建议：[自拍 / 网络搜索关键词 / 素材库]
时长：[建议秒数]
```

规则：
- 描述要具体到"搜什么关键词能找到"
- 不写拍摄指令（创作者自己决定怎么拍）
- 标注"切入时机"（旁白说到哪句话时切过去）

### C · 纯脚本
跳过本步骤，直接告知创作者："此频道为纯脚本模式，分镜和剪辑方案步骤已跳过。脚本完成即可进入录制。"

---

## 四、输出（交付物）

```
04_Storyboard/[标题]/[标题]_YYYYMMDD.md
```

分镜文件格式（AI生图模式）：

```markdown
# 分镜 — [标题]

视觉风格：写实摄影，16:9，1080p

---

[原文：旁白原文，保持不动]

Prompt 01：[静态画面描述]
运动：[Ken Burns缓慢推进 / I2V（具体动作） / 静止]

---
```

---

## 五、核心规则（不可违反）

1. **Prompt 自给自足**：每条 Prompt 无相对引用（禁止"同上"、"比上一个"）
2. **时空锚点必须**：每条 Prompt 含①时代②地域文化，不依赖上下文
3. **静态描述**：描述凝固状态，不是动作过程
4. **禁止文字入镜**：Prompt 中不描述图片内的中文文字（英文可以）
5. **Naming 段**：运动字段写"静止"，时长 ×1.5
6. **视觉盲点**：找不到画面标注 `⚠️ 视觉盲点`，最后汇总

**写入方式**：必须用 Bash `cat >` heredoc 写入，禁止用 Write/Edit 工具（分镜文件通常 500-1200 行）

---

## 六、双层 Review 清单

### 🤖 Layer 1 — AI 自检项

- [ ] 无任何相对引用？
- [ ] 每条 Prompt 有时代 + 地域？
- [ ] 所有描述是静态状态？
- [ ] Naming 段运动字段是"静止"？
- [ ] 视觉盲点已标注并汇总？

### 👤 Layer 2 — 创作者判断项

- 快速翻一遍 Prompt 列表：有没有描述模糊到你脑子里看不到具体画面的？
- Hook 段排列起来，节奏是否够快？
- 视觉盲点列表：每一条，你给出处理决定（替换旁白 / 跳过 / 用替代图）
