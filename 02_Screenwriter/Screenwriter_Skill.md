# 编剧 Skill 文件

**角色**：Screenwriter（编剧）
**对应流程**：STEP 2 — 脚本开发
**Review 节点**：REVIEW-2（脚本审核）

---

## 一、角色定义

编剧的工作是把选题的核心逻辑，转化为一个**观众能真实经历的叙事序列**。

不是写大纲，不是写摘要。是写一个让人从头看到尾、每隔30秒就有一次"啊原来如此"的脚本。

在制片厂里，这是 AI **部分替代**的角色——AI 可以生成结构和语言，但"这句话是不是真的在观众脑子里打开了什么"，需要你来判断。

---

## 二、输入

- 选题文档（STEP 1 输出）
- 制片人 Greenlight 批注
- 触发词：`写脚本` / `生成脚本 [选题文件路径]`

---

## 三、输出（交付物）

**第一阶段 — 脚本定稿**（`写脚本` 触发）

```
02_Scripts/YYYY-M-D_标题/
└── 脚本名_v1.md           ← 主脚本（干净版，无停顿标记）
```

经创作者明确确认定稿后，才进入第二阶段。

**第二阶段 — 配套文件**（`生成配套文件` 触发）

```
02_Scripts/YYYY-M-D_标题/
├── 脚本名_v1_TTS.txt      ← TTS 版（含 <#0.25#> 停顿标记，克制使用）
├── 脚本名_v1_EN_TTS.txt   ← 英文 TTS 版
├── 封面描述_v1.txt        ← 国内平台封面
├── 封面描述_YouTube_v1.txt
├── 视频简介_v1.txt
└── 视频简介_YouTube_v1.txt
```

## 四、工作规则

### 结构模板（从 Channel_Config.md 读取）

执行时读取 `Channel_Config.md` 中的「脚本结构模板」字段，不同频道类型有不同的结构。

> **示例**（以知识讲解类频道为例，其他类型请参见对应的 Config 模板）：
>
> ```
> Hook（前10秒）
>   → 0-3秒：快速剪辑入口（具体实物或历史场景）
>   → 3-5秒：字幕爆炸（核心悬念）
>   → 5-10秒：权威背书
> Contrast（对比段）
> Naming（命名时刻）
> Mechanism（机制段，2-3轮）
> Mapping（个体迁移）
> Closing（收束）
> ```
>
> 历史故事类使用 Scene→Identity→Conflict→Climax→Punchline→Series Hook
> 电影解说类使用 Hook→Synopsis→Insight→Verdict→CTA

### [频道专属语气规则 — 从 Channel_Config.md 读取]

执行时读取 `Channel_Config.md` 中的「语气标准」字段。

> **示例**（以“理性避难所”频道为例，仅供参考）：
> - 禁止共情句："我们都经历过……" "相信很多人……"
> - 禁止公式句："接下来我要……" "映射到你身上……"
> - 禁止解决方案（认知收束类选题）：不给建议，给冷感判断
> - CTA 固定结尾：关注理性避难所。每期一个框架，帮你把世界看得更清楚一点。

### [通用规则 — 所有频道适用]

- 停顿标记只写在 `_TTS.txt` 文件，`.md` 脚本保持干净
- 停顿时长统一用 `<#0.25#>`，克制使用：仅在**重要句末**或**需要强调的节点**添加，普通句末不加停顿；全篇停顿标记宜少不宜多
- 字数范围：从 `Channel_Config.md` 读取（不同频道不同）
- 路径验证：脚本必须保存在 `02_Scripts/` 下，不写入其他目录

### 视频简介格式规范

`视频简介_v1.txt` 和 `视频简介_YouTube_v1.txt` 均须包含以下结构：

```
《标题》

候选标题 1：[标题]
候选标题 2：[标题]
候选标题 3：[标题]

---

[正文简介内容]
[CTA]
```

**标题初稿规则**（此处为编剧草拟版，Distribution 步骤会进一步精化）：
- 提供 3 个候选标题，三者采用不同的钩子策略
- 每个标题制造"悬而未决的问题"——读完标题，观众脑子里要有一个疑问在悬着
- 禁止使用"揭秘" / "震惊" / "万万没想到"等流量词
- 从脚本的 Hook 段和 Naming 时刻提炼标题素材（最强的那句话，就是标题原料）

## 五、双层 Review 清单

### 🤖 Layer 1 — AI 自检项

- [ ] Hook 开场符合选题文档标注的谜题类型（Wow → 极端事实开场，不直接提问；Why → 呈现反常识结果，不给原因；What if → 第一人称场景）？
- [ ] 读完开头，读者脑子里有没有一个还没被回答的问题？有 → 合格；没有 → 重写。
- [ ] Hook 开场是具体实物或历史场景（非抽象概念）？
- [ ] 六大结构模块均可识别：Hook / Contrast / Naming / Mechanism / Mapping / Closing
- [ ] Naming 时刻存在，且有白话解释（非定义，是"换句话说"）？
- [ ] Mechanism 段 ≥ 2 个案例，且跨领域？
- [ ] 收束类型与选题类型匹配（揭示系统规律 → 认知；有操作空间 → 行动）？
- [ ] 停顿标记只在 `_TTS.txt`，主脚本 `.md` 无标记？
- [ ] 停顿标记均为 `<#0.25#>`，无其他时长？停顿位置是重要句末或强调节点，非每句末都有？
- [ ] **内容安全检查**（参见 `_Templates/Content_RedLines.md`）：无禁忌内容（毒品 / 疫情 / 政治 / 宗教 / 分裂）？举例以美/欧/日为主？如触碰红线 → 立即停止，替换案例后重写，不继续后续流程。
- [ ] 无共情说教句（"我们都经历过" / "你应该" 等）？
- [ ] 无公式句（"接下来映射到你身上" 等显式宣布动作）？
- [ ] 视频简介文档顶部有《标题》区块，含 3 个候选标题？
- [ ] 三个候选标题使用了不同的钩子策略（非同质化）？
- [ ] 所有文件路径以 `02_Scripts/` 开头？

### 👤 Layer 2 — 创作者判断项

- 从头看一遍脚本，每隔 30 秒有没有一次"啊原来如此"的时刻？
- 脚本里最强的那一句是哪一句？你能立刻指出来吗？
- 语气听起来像"冷静呈现事实的人"，还是像"想让你认同我的人"？
- **Kennedy 最终问题**：发出去，你的老粉会觉得"这就是他"还是"他变了"？

## 六、常见错误

| 错误 | 表现 | 修正 |
|------|------|------|
| Hook 太抽象 | "经济学里有一个有趣的规律" | "1970年，英国政府宣布要解决能源危机" |
| Wow 谜题变成 Why 谜题 | 极端事实后面跟了一句"为什么会这样？" | 让事实自己说话，删掉提问句，让观众自己发问 |
| 命名时刻缺失 | 讲了规律但没给它起名字 | 找一个观众能记住的词，哪怕是临时造的 |
| Mapping 显式宣布 | "接下来把它映射到你身上" | 直接用一个第二人称场景，不宣布动作 |
| 认知收束给了解决方案 | "所以你应该…" | 冷感判断："这个机制不会因为你知道它而停止运作" |
| 停顿标记污染主脚本 | `.md` 里出现 `<#0.25#>` | 只在 `_TTS.txt` 里写，重新生成干净版 |
| 停顿过密 | 每句末都加停顿 | 删掉非重要节点的停顿，仅保留重要句末或强调点 |
