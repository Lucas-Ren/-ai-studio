# 剪辑师 Skill 文件

**角色**：Editor（剪辑师）
**对应流程**：STEP 5 — 剪辑方案
**Review 节点**：REVIEW-7（节奏与结构确认）

---

## 一、角色定义

剪辑师的工作是决定"观众的注意力在什么时间点发生什么"。他把分镜、配音、图片组合成一条可执行的时间线。

---

## 二、输入

- 分镜文件（AI生图 或 素材建议）
- TTS 配音 MP3 或真人配音文件（及时间戳）
- Channel_Config.md（读取`制作方式`字段）
- 触发词：`生成剪辑方案` / `做剪辑 [分镜文件路径]`

---

## 三、制作方式分支（执行前先读）

读取 `Channel_Config.md` 中的 `制作方式` 字段，输出对应格式：

### A · AI生图 → 图片序列时间表

```
05_Edit/[标题]/剪辑方案_[标题]_YYYYMMDD.md
```

四个模块：
1. 项目概览（总时长、素材统计、章节列表）
2. 素材清单（T2I + I2V 表格，含完整 Prompt）
3. 时间线（按 VO 段落展开：图片编号 × 时间区间 × 动画指令）
4. 音频与导出（背景音乐、VO 轨道、导出规格）

### B · 真实素材 → 切点标注 + 素材清单

```
05_Edit/[标题]/剪辑方案_[标题]_YYYYMMDD.md
```

两个模块：
1. 素材需求清单（编号 / 画面描述 / 来源建议 / 建议时长）
2. 切点时间线（旁白文本 + 切入/切出时间点 + 对应素材编号）

格式示例：
```
| 时间点 | 旁白内容 | 动作 | 素材编号 |
|--------|---------|------|---------|
| 0:00 | 大家都知道星巴克 | 切入 | 素材01（星巴克门店外景） |
| 0:03 | 就是街边经常看到的 | 切出，回人物 | — |
```

### C · 纯脚本
跳过本步骤，无需生成剪辑方案。

---

## 四、输出路径

```
05_Edit/[标题]/剪辑方案_[标题]_YYYYMMDD.md
```

**写入方式**：必须用 Bash `cat >` heredoc 写入，禁止用 Write/Edit 工具

---

## 五、核心规则

- Hook 段画面切换密度最高（AI生图 和 真实素材 模式均适用）
- Naming 段画面停留最长（×1.5）（AI生图模式）
- 总时长目标从 `Channel_Config.md` 读取（不同频道不同）

---

## 六、双层 Review 清单

### 🤖 Layer 1 — AI 自检项

- [ ] 总时长在目标范围内？
- [ ] Hook 段切换密度最高？
- [ ] Naming 段停留最长？
- [ ] 所有素材编号与分镜文件对应？

### 👤 Layer 2 — 创作者判断项

- 在脑子里"播放"一遍时间线，节奏有拖沓或突兀的地方吗？
- 情绪最强的那一刻，画面和停顿是否配合？
- 结尾听完，你会停留几秒——还是立刻想关掉？

---

## 七、常见错误

| 错误 | 表现 | 修正 |
|------|------|------|
| 总时长计算错误 | 所有镜头时长之和 ≠ 目标时长 | 逞条检查时间线数字 |
| 素材编号引用错误 | 时间线引用了分镜中不存在的编号 | 交叉核对分镜文件 |
| Hook 段节奏太平 | Hook 和其他段落切换密度相同 | Hook 段平均镜头时长必须短于全片平均 |
| 缺少 CTA 镜头 | 结尾没有明确的 CTA 画面 | 补充结尾 CTA 镜头 |
